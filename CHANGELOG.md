- 빈 운동 카탈로그에서도 picker를 열 수 있도록 `onCreateExercise` 제공 시 활성화.
- 사진 3장 이상인 운동을 이미지 공유할 때, 첫 2장만 포함된다는 확인 다이얼로그 노출.

### 변경
- 기록·루틴·수업 상세의 읽기 전용 운동/세트 목록을 공용 `RoutineReadonly` 컴포넌트로 통합
  (그동안 세 페이지가 동일 마크업을 복붙). 세트 값은 숫자 강조 + 단위(kg·×·회) 약화로
  대비를 줘서 가독성 개선(`renderSetValue`, `routine-readonly--emph`).
- 운동 상세 화면에 collapsing 헤더 적용: 스크롤하면 사진이 줄고(`heroH = 풀높이 − 스크롤량`)
  제목이 사진 하단 모서리를 타고 올라와, 사진이 완전히 접히면 상단(바 아래)에 sticky 고정.
  사진 풀높이는 `window.innerWidth × 3/4`로 계산.
- 운동 상세 "최근 기록"과 운동 기록 전체(`/exercises/:id/history`) 표시 개선:
  세트를 칩 단위로 표시(축약 `formatSetShort`, 예 `77.5kg×10`), 날짜는 요일 없는
  `formatShortDate`로 칩과 같은 줄에 인라인 배치. 최근 기록은 3개까지만 노출(초과 시 더보기).
- 데이터 관리 화면의 "데이터 초기화"를 화면 하단에 구분선 + 작은 텍스트 버튼으로 약화
  (섹션 타이틀/경고문구 제거). 실수 방지는 기존 확인 다이얼로그가 담당.
- 데이터 복원 경고문구의 빨간 강조 제거 + 문구 완화.
- `btn--ghost` 기본 배경을 surface로 변경(페이지 배경 위 보조 버튼의 가시성 개선,
  가장 흔한 컨텍스트를 기본값으로). 다이얼로그(배경 surface) 안에서는
  `.dialog .btn--ghost`로 `--color-bg` 채움 오버라이드해 취소 버튼 대비 유지.
- 공유 이미지 레이아웃: 사진 2장은 좌우 분할. 좌측 컬럼을 사진 2장으로 카드
  위/아래/왼쪽 가장자리까지 풀블리드로 채우고(cover), 우측에 운동명·메타·설명·푸터 배치.
  3장 적층 시 세로로 과도하게 길어지던 문제를 해결하기 위해 공유 사진을 최대 2장으로 제한.

### 수정
- 캐러셀 페이지 점이 흰색 사진 위에서 보이지 않던 문제(scrim 추가).

## [0.1.1] - 2026-06-15

### 수정
- SPA 라우트 새로고침/딥링크 시 404 발생 문제. Vercel rewrite + Service Worker
  `navigateFallback: index.html` 적용.
- IndexedDB 데이터 유실 예방: `navigator.storage.persist()`로 영구 저장소 요청.
- 폼 저장 실패가 조용히 무시되던 문제: 저장 핸들러를 try/catch로 감싸 "저장 실패" 토스트 노출.
- Android 설치 불가 문제: PWA 설치 기준 충족을 위해 PNG 아이콘(192/512/maskable) 추가.
